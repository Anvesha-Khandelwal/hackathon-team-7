OS Synchronisation — Semaphore & Race Condition CONCEPTS
⚡ Race Condition (No Semaphore)
Two threads write to the same shared board cell simultaneously with no protection. The second write overwrites the first — board state becomes undefined.
Thread-1: read cell[6] → '.'
Thread-2: read cell[6] → '.' ← same!
Thread-1: write cell[6] = 'X'
Thread-2: write cell[6] = 'O' ← OVERWRITE
Result: CORRUPTED ⚠
🔒 Semaphore Protected (Mutex)
A binary semaphore (S=1) acts as a mutex. P() decrements S to 0 (lock). V() increments S to 1 (unlock). Only one thread in critical section at a time.
Thread-1: P(S): S=1→0 ✓ ACQUIRED
Thread-2: P(S): S=0 → BLOCKED ⏳
Thread-1: write safely → V(S): S=0→1
Thread-2: unblocked → P(S): S=1→0
Result: CONSISTENT ✅
